# 包子智能代理 - 整体联动架构 (v2.0 细化版)

> 所有功能一环扣一环，形成完整循环

---

## 一、整体联动图

```
用户消息
    ↓
① 分析内容 → 调用 smart-link.js → 写入记忆
    ↓
② 搜索 → memory_search（不知道的事）
    ↓
③ 执行 → 工具调用
    ↓
④ 错误？ → 立即写 ERRORS.md
    ↓
⑤ 回复用户 → 更新状态
    ↓
⑥ 心跳（15分钟）→ 检查待办/成长
    ↓
循环回到①
```

---

## 二、各模块详细联动

### 1. 记忆系统 ↔ 搜索
| 动作 | 结果 |
|------|------|
| 写入记忆 | 自动更新 index.json |
| 搜索时 | 先读索引 → 快速定位 |

### 2. 记忆系统 ↔ 自我改进
| 动作 | 结果 |
|------|------|
| 错误发生 | 立即写入 ERRORS.md |
| 心跳时 | 读取错误 → 分析模式 → 更新 LEARNINGS.md |

### 3. 记忆系统 ↔ 成长循环
| 动作 | 结果 |
|------|------|
| 任务创建 | 写入 SESSION-STATE.md |
| 心跳时 | 检查完成情况 → 更新 outcome-journal.md |
| 定期 | 分析模式 → 更新 recurring-patterns.md |

### 4. 搜索 ↔ 主动预测
| 动作 | 结果 |
|------|------|
| 搜索结果 | 作为上下文 |
| 主动预测 | 基于历史记忆 → 提前准备 |

## 三、工作流程（必须按顺序）

### 响应流程（每次）
```
1. 收到消息
2. 分析内容
3. 调用 smart-link.js → 自动分类写入
4. 如果"不知道" → memory_search
5. 执行任务
6. 错误？ → 写 ERRORS.md
7. 回复用户
8. 更新 SESSION-STATE.md 状态
```

### 心跳流程（15分钟）
```
1. 读取今日日记
2. 检查 SESSION-STATE.md 待办
3. 更新 outcome-journal.md
4. 分析 recurring-patterns.md
5. 搜索外部信息（如需要）
6. 回归正常工作
```

## 四、强制规则（必须执行！）

| 规则 | 触发条件 | 动作 |
|------|---------|------|
| 不知道先搜 | 遇到不确定的事 | memory_search |
| 错误立即记 | tool执行失败 | 写 ERRORS.md |
| 纠正立即记 | 用户否定/纠正 | 写 LEARNINGS.md |
| 重要立即记 | 用户给任务/偏好 | 写 SESSION-STATE.md |
| **每次响应** | **所有消息** | **smart-link.js** |
| **回复后第一件事** | 收到消息 | **调用smart-link.js** |
| 每次心跳 | 15分钟 | 检查待办 |

**核心：每次回复后第一件事就是调用 smart-link.js！**

## 五、心跳流程（15分钟一次）

### 每次心跳执行三个功能

```
定时触发（15分钟）
    ↓
① 状态检查
    ↓
② 联动整理
    ↓
③ 主动记忆
    ↓
回归正常工作
```

### 1. 状态检查
- 检查系统是否正常
- 检查待办任务
- 检查是否有异常

### 2. 联动整理
- 读取今日日记
- 检查是否有遗漏
- 更新index索引
- 清理temp目录

### 3. 主动记忆
- 检查重要待办
- 想起重要的事提醒用户
- 检查未完成的任务

### 主动记忆例子
- "张先生，上次说的原油止盈改为4美元还没改呢"
- "记得昨天交易亏损了吗？要不要分析一下？"

## 六、关键触发词

### 自动分类关键词
```
错误/失败 → ERRORS.md
纠正/否定 → LEARNINGS.md  
任务/待办 → SESSION-STATE.md
偏好/设置 → USER.md
重要/关键 → important/
其他 → diary/
```

---

## 七、核心原则

**一环扣一环**：
- 任何消息 → 先检查记忆
- 任何错误 → 立即记录
- 任何任务 → 更新状态
- 任何回复 → 确认记录

**宁可多记，不要漏记！**
